## 1. 目的

- 今まで Excel で手集計していた FPL のデータ収集・集計を自動化する
- そのデータを **動画スライド・X投稿の素材**としてすぐ使える形で1か所に表示する
- 大衆にも分かるよう、専門用語は避けるか日本語に言い換える

## 2. 技術構成（シンプル最優先）

- **データ収集**: Python スクリプト1本。公式APIを叩いて結果を JSON ファイルに保存
- **表示**: 静的サイト（HTML / CSS / 素のJavaScript）。保存した JSON を読んで表に描画
  - フレームワークは不要。初心者がいじりやすいシンプルな構成にする
- **ホスティング**: 無料（GitHub Pages を推奨。Claude Code がセットアップまで行う）
- **更新方法**: 週1回、収集スクリプトを実行 → 生成された JSON をコミット&プッシュ
  - この「週1の更新コマンド」を1行で実行できるようにまとめておく（例: シェルスクリプト or npm script）

サーバーやデータベースは使わない（コストと管理を増やさないため）。

## 3. データソース（公式 FPL API・無料・認証不要）

ベースURL: `https://fantasy.premierleague.com/api/`

主に使うエンドポイント（※フィールド名は実際のレスポンスを見て確認・マッピングすること）:

- `bootstrap-static/` … 選手一覧(elements)、チーム(teams)、節情報(events)、ポジション(element_types)
- `fixtures/` … 全試合の日程・結果・難易度
- `element-summary/{選手ID}/` … 選手ごとの詳細履歴（必要に応じて）
- `entry/906697/` … マイチーム（FPL侍のチーム）の基本情報
- `entry/906697/event/{節ID}/picks/` … 指定した節のマイチーム15人

実装メモ:
- リクエスト時は丁寧に（適切な User-Agent を付ける、連続アクセスは間隔を空ける）
- API が落ちている/構造が変わった場合に備え、エラー処理と「前回データの保持」を入れる
- 取得した生データは `data/raw/` に、集計後は `data/site/` に分けて保存すると管理が楽

## 4. v1（最初に作る）で表示するもの

1. **チーム別 無失点率ランキング**
   - 計算: 各チームの「無失点だった試合数 ÷ 消化済み試合数」（fixtures の結果から算出）
   - 用途: 動画「無失点率」コーナー、X の月曜投稿
2. **選手別 ランキング**（得点・アシスト・得点期待度=xG/xGI）
   - bootstrap-static の elements から集計。ポジション・チーム名は日本語表示用に変換
   - 用途: おすすめ選手、X のデータ投稿
3. **次節の日程＋相手の強さ**
   - 「次の節」の試合を一覧化。難易度は数字ではなく「弱い/普通/強い」など言葉で表示
   - 用途: 動画「次節日程」コーナー
4. **おすすめ選手（簡易版）**
   - 出場時間が一定以上の選手を、調子(form)や得点期待度で並べ替えて上位を表示
   - まずは単純な並べ替えでOK（高度なロジックは後回し）

### v2 以降（後で追加）

- **マイチーム表示**（entry/906697 の現在の15人 → 移籍プランの土台）
- 画像書き出し（動画スライド・X用に表をそのまま画像化）
- 自動更新（定期実行の仕組み）

## 5. 表示・デザインの方針

- **スマホ最優先**（縦画面で表が見やすいこと）
- 表示テキストはすべて**日本語**。専門用語は言い換える
  - 例: clean sheet → 無失点 / xG・xGI → 得点期待度 / fixture → 日程 / difficulty → 相手の強さ
- ランキング表は「順位・名前・チーム・数値」が一目で分かるシンプルな見た目
- 各表に「最終更新日（取得した節）」を必ず表示する

## 6. ディレクトリ構成（提案。Claude Code が最適化してよい）

```
fpl-samurai-site/
├─ scripts/
│   └─ update.py        # APIから取得→集計→JSON生成
├─ data/
│   ├─ raw/             # APIの生データ
│   └─ site/            # サイトが読む集計済みJSON
├─ public/              # 公開する静的サイト
│   ├─ index.html
│   ├─ style.css
│   └─ app.js
├─ update.sh            # 「週1更新」を1コマンドで実行
└─ README.md            # 初心者向けの使い方（日本語）
```

## 7. 週1の更新フロー（README に日本語で明記してもらう）

1. ターミナルで `./update.sh` を実行（中で update.py が走ってデータ更新）
2. 生成された JSON を git でコミット&プッシュ
3. 数分後、公開サイトに反映される

→ この1〜3を初心者がコピペで実行できるよう、コマンドと手順を README にまとめること。

## 8. 環境の前提

- Claude Code が使える状態（インストールは公式手順に従う）
  - 参考: https://docs.claude.com/en/docs/claude-code/overview
- Python が使えること（バージョン確認も Claude Code に任せる）
- GitHub アカウント（GitHub Pages 公開用。無ければ作成手順も案内してもらう）

## 9. 完成の定義（v1）

- `./update.sh` 一発で最新データに更新できる
- 公開URLにアクセスすると、上記4つの表がスマホで見やすく日本語表示される
- 各表に最終更新（節）が出ている
- README を見れば、初心者でも毎週の更新ができる
