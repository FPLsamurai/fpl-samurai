# ⚔️ FPL侍 データサイト

Fantasy Premier League（FPL）の公式データを自動で取得して、
**動画スライドやX投稿の素材**としてすぐ使える形で表示する静的サイトです。

サイトは「選手」「チーム」「次節」の3つのタブに分かれています。

**選手タブ**
- 全試合・直近3／5／10試合（高機能テーブル・全タブ同じ列構成）。列（左から）：
  順位・写真・選手名・チーム・POS・コスト・ポイント・コスパ（ポイント÷コスト）・所持率・
  ゴール・アシスト・無失点数・スタメン数・出場時間・xG・xG/90・G-xG（上振れ／下振れ）・xA・xA/90・
  DEFCON・DEFCON/90・ボーナス・Pts/試合・セーブ・セーブ/90・PKストップ・イエロー・レッド・
  次の3試合（相手名と、その試合の得点期待値＝選手のxG/90×相手の守備の弱さ。オフシーズン中は「—」）
  - 直近3／5／10タブは「チームの直近N試合すべて」で集計（ベンチ・欠場の0分も1試合として数えます）
  - 「チーム」列のドロップダウンで、特定チームの選手だけに絞り込めます
  - **並べ替え**：見出しをタップ（↑昇順／↓降順）
  - **絞り込み**：各列の枠に数値を入れて「◯◯以上／以下」、選手名は検索、ポジションは選択
  - **固定列**：左に固定する範囲は ⚙パネルで選べます（固定しない〜ポイントまで。標準は選手まで）。表はページ全幅
  - **動画用の列設定**：「⚙ 列の表示・並び替え」で、いらない列を隠す／順番を入れ替え可能。
    設定はブラウザに保存され、次回も同じ表示になります（「標準に戻す」でリセット）
  - 写真はFPL公式（今シーズン版）の画像を自動表示
  - 選手名のカタカナ：登録した選手だけ、英語名の下にチーム名と同じ色で表示
    （登録は `data/japanese_names.json` に追記。下の「選手名のカタカナを増やす」参照）
- ホーム / アウェイ別の成績
- セットプレー担当者DB（PK・直接FK・CKの担当を優先順位つきでチーム別に表示。公式データ）

**チームタブ**
- 合計（xG=攻撃の期待値、被xG=守備で許した期待値）
- 節別（チームを選ぶと節ごとのxG・被xG）
- 無失点率ランキング
- 直近フォーム（直近5・10試合の平均xG / 被xG）

**次節タブ**
- 予測：クリーンシート率（無失点で終える確率）とゴール期待値
  - 計算：ポアソン分布。相手と自チームの直近xG・被xGを、リーグ平均xG（μ）で割って算出
- 日程：次節の試合と「相手の強さ」（弱い／普通／強い）

**マイチームタブ**
- FPL ID を入力すると、そのチームの「スカッド15人（キャプテン・ベンチ・チーム価値）」と
  「参加ミニリーグの順位（順位表ボタンで上位20も表示）」を見られます
- IDの調べ方：FPL公式にログイン →「Points」ページのURLにある `/entry/数字/` の数字
- 入力したIDはブラウザに記憶されます（次回から入力済み）
- 仕組みの注意：FPL公式APIはブラウザから直接呼べないため、無料の中継サービス
  （allorigins.win）経由で取得しています。中継が混んでいると数秒〜十数秒かかったり、
  まれに失敗することがあります（時間をおいて再度押せばOK）。送られるのは入力したID
  （＝FPL上で誰でも見られる公開情報）だけです

> 用語：xG/xGI=得点（＋アシスト）の期待度、被xG=守備で相手に与えた得点期待度（少ないほど良い）。

## 📁 中身の説明（ざっくり）

```
FPLsite/
├─ scripts/
│   ├─ update.py     ← データを取りに行って集計するプログラム
│   └─ serve.py      ← 自分のPCで表示を確認するための小さなサーバー
├─ data/
│   ├─ raw/          ← 取ってきたままの生データ（自動生成・Gitには載せません）
│   │   └─ players/  ← 選手ごとの試合データ（自動保存。次回更新を速くします）
│   ├─ japanese_names.json ← 選手名のカタカナ対応表（自分で追記できます）
│   └─ site/         ← サイトが使う集計済みデータ
├─ public/           ← ★公開されるサイト本体（これがネットに出ます）
│   ├─ index.html    ← ページの中身
│   ├─ style.css     ← 見た目（色・レイアウト）
│   ├─ app.js        ← データを表に描く動き
│   └─ data.json     ← サイトが読むデータ（update.pyが自動でコピー）
├─ update.sh         ← ★週1回の更新コマンド（これ1つでOK）
└─ README.md         ← この説明書
```

> 普段いじるのは `public/` の中（見た目を変えたいとき）くらいです。
> データの更新は `./update.sh` を実行するだけです。

## 🖥 自分のPCでサイトを確認する

**※「ファイルをダブルクリックで開く」ではデータが表示されません。** 下の手順で開いてください。

1. ターミナル（Macの「ターミナル」アプリ）を開く
2. このフォルダに移動する（フォルダをターミナルにドラッグすると楽です）:
   ```
   cd "/Users/ozawakoutarou/Downloads/FPLsite"
   ```
3. 確認用サーバーを起動する:
   ```
   python3 scripts/serve.py
   ```
4. ブラウザ（Safari等）で次のアドレスを開く:
   ```
   http://localhost:8000
   ```
5. 確認が終わったら、ターミナルで `Ctrl` + `C` を押すとサーバーが止まります。

## 🔄 毎週の更新のやり方（一番大事）

毎週これをやるだけです:

1. ターミナルでこのフォルダに移動（上の手順2と同じ）
2. 次のコマンドを実行:
   ```
   ./update.sh
   ```

これで「最新データの取得 → 集計 → （GitHub設定済みなら）公開サイトへ反映」まで自動で行います。
公開設定がまだのときは、データの更新だけ行い、次にやることを画面に表示します。

> **取得には数分かかります。** 選手1人ずつの試合データを公式APIから集めるためです
> （途中で止めても大丈夫。同じ日にもう一度実行すると、続きから速く終わります）。
> 公開サイトへの反映も、プッシュ後さらに数分かかります。少し待ってから公開URLを開いてください。

## 🌐 はじめての公開（GitHub Pages・無料）
ここは**最初の1回だけ**やる作業です。一度設定すれば、以降は `./update.sh` だけで更新できます。

### 手順A：GitHubアカウントを作る（持っていない場合）

1. https://github.com/signup をブラウザで開く
2. メールアドレス・パスワード・ユーザー名を入力してアカウントを作成
   - ⚠ パスワードの入力はご自身で行ってください
3. 確認メールが届くので、メール内のボタンで認証する

### 手順B：新しい置き場所（リポジトリ）を作る

1. ログインした状態で https://github.com/new を開く
2. **Repository name** に好きな名前を入力（例: `fpl-samurai`）
3. **Public（公開）** を選ぶ ※GitHub Pagesの無料公開にはPublicが必要です
4. 他はそのままで「Create repository」を押す
5. 作成後の画面に出る **あなたのリポジトリのアドレス**（例:
   `https://github.com/ユーザー名/fpl-samurai.git`）を控えておく

### 手順C：このフォルダをGitHubにアップロードする

ターミナルでこのフォルダに移動し、次を順番に実行します
（`ユーザー名` と `fpl-samurai` はご自身のものに置き換えてください）:

```
git add .
git commit -m "最初の公開"
git branch -M main
git remote add origin https://github.com/ユーザー名/fpl-samurai.git
git push -u origin main
```

- 途中でGitHubのユーザー名やパスワードを聞かれたら入力してください
  （パスワードの代わりに「トークン」を求められる場合があります。
  その場合は GitHub の画面 Settings → Developer settings →
  Personal access tokens から作成して使います）

### 手順D：GitHub Pages を「オン」にする

1. ブラウザでご自身のリポジトリを開く
2. 上の「**Settings**」→ 左メニューの「**Pages**」を開く
3. 「Build and deployment」の **Source** を「**GitHub Actions**」に変更
4. これで設定完了です

数分後、`https://ユーザー名.github.io/fpl-samurai/` で公開サイトが見られます。
（このアドレスがあなたの公開URLです。X や動画の概要欄に使えます）

## 🈁 選手名のカタカナを増やす

公式APIに日本語名が無いため、`data/japanese_names.json` に書いた選手だけカタカナが表示されます。

1. `data/japanese_names.json` をテキストエディタで開く
2. 行を追加する（左がFPLの英語表記、右がカタカナ）。英語表記は表の選手名と同じにする：
   ```
   "Salah": "サラー",
   "Saka": "サカ",
   ```
   ※ 各行の最後はカンマ「,」。一番最後の行だけカンマを付けない、が書式のルールです。
3. 保存して `python3 scripts/update.py` を実行すると反映されます。

> 英語表記が分からないときは、サイトの選手名（英語）をそのままコピーして左側に貼ればOKです。

## ❓ 困ったときは

- **表示が真っ白／表が出ない（自分のPC）**
  → ファイルを直接開いていませんか？「自分のPCでサイトを確認する」の手順で
  `python3 scripts/serve.py` を使って開いてください。

- **`./update.sh` が「permission denied」と出る**
  → 一度だけ次を実行してください: `chmod +x update.sh`

- **データが古い気がする**
  → APIが一時的に不調だと、前回のデータを表示します（画面上部に黄色い注意が出ます）。
  時間をおいて `./update.sh` をもう一度実行してください。

- **公開URLが「404」になる**
  → 手順Dの Pages 設定（Source = GitHub Actions）が済んでいるか、
  プッシュから数分待ったか確認してください。
